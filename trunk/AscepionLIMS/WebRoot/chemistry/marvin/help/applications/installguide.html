Windows Instructions

    Instructions

    1. After downloading, double-click on marvinbeans-VERSION-windows.exe or marvinbeans-VERSION-windows_with_jre.exe.
    2. After the installation, you can add the bin folder of Marvin Beans to the PATH to be able to run Marvin applications from any directory in the command line. For more details about the editing the PATH, please consult the Windows Help.

    Notes

    • If you do not have a Java installed, be sure to download the package above which includes it.
    • If you are not sure that Java is installed, you can check it by the following way:
      1. Select Command Prompt from the Accessories sub-menu in the Start menu.
      2. Type the following commands in the opened Command Prompt window: java -version
      3. You will get the following error message if Java is not available on your machine: 
        'java' is not recognized as internal or external command, operable program or batch file
         If Java has been installed, the version number of Java will be printed: 
        java version "1.5.0_11"
Java(TM) 2 Runtime Environment, Standard Edition (build 1.5.0_11-b03)
Java HotSpot(TM) Client VM (build 1.5.0_11-b03, mixed mode, sharing)

Mac OS X Instructions

    Instructions

    • After downloading, double-click marvinbeans-VERSION-macos.dmg.
    • After the installation process, add the bin folder of the Marvin Beans folder to the PATH to be able to run Marvin applications from any directory in command line.

    Notes

    • Requires Mac OS X 10.0 or later
    • The compressed installer should be recognized by Stuffit Expander and should automatically be expanded after downloading. If it is not expanded, you can expand it manually using StuffIt Expander 6.0 or later.
    • If you have any problems launching the installer once it has been expanded, make sure that the compressed installer was expanded using Stuffit Expander. If you continue to have problems, please contact technical support.

Linux / Solaris Instructions

    Instructions

    1. After downloading, open a shell and cd to the directory where you downloaded the installer.
    2. At the prompt type: ./marvinbeans-VERSION-linux.sh
    3. After the installation, add the bin subdirectory of the Marvin Beans directory to the PATH to be able to run Marvin applications from any directory.

    Notes

    • You need to install a Sun's Java Runtime Environment 1.4.2 (or later). You can download it from Sun's Java web site or contact your OS manufacturer.
    • If the installer does not start, check whether JAVA_HOME/bin is in PATH (where JAVA_HOME is the directory of Java).
      To chech it, type the "which java" command that shows the location of the java launcher. You should get something like this: 
      /usr/java/j2sdk1.4.2_04/bin/java
      If java is missing from PATH, you will see something like that: 
      /usr/bin/which: no java in (/usr/java/j2sdk1.4.2_04/bin:/opt/apache-ant-1.6.1/bin:/usr/kerberos/bin:/usr/local/bin:/bin:/usr/bin:/usr/X11R6/bin:/home/vertset/bin)

Instructions for All Other Platforms

    Instructions

    • After downloading go to the directory where you downloaded marvinbeans-VERSION.zip then uncompress the zip file.
    • You can start applications by script or batch files, which you can find in the marvinbeans/bin directory.

    Notes

    • Requires Java 1.4.2 (or later). You can download it from Sun's Java web site or contact your OS manufacturer.
    • You need an expander which can handle zip extension.
    • Batch files (bin/*.bat) have to be initialized before first use. Edit files to set the MARVINBEANSHOME variable to the full path of the directory where Marvin Beans is located.

Howto uninstall

    Use the uninstaller to remove Marvin Beans from your machine.

    Windows

    Double click on uninstall.exe in the Marvin Beans's home folder or select Marvin Beans from the Add / Remove programs list on Control Panel.

    OS X

    Double click on the uninstall in the Marvin Beans' home directory.

    Linux / Solaris

    Launch the uninstall script in theMarvin Beans' home directory.

Instructions for the additional package

Who needs this package?

Install marvinbeans-lib-VERSION-signed.zip only that case if you need the signed version of Marvin Beans package. In most cases, the non-signed Marvin Beans package is enought for you. If you would like to launch Marvin applications via Java Web Start from your server, you need the signed version. (Because of security reason, web start accept only signed jars.) This archive does not contains the whole Marvin Beans package. It does not work alone. It requires an installed Marvin Beans package.

Installation

  • Check the version ID of Marvin Beans package that you have already installed (if not complete it first). In the help of your Marvin Beans package, you can find the version ID of your installed disribution. The "version.properties" file also includes the version ID. (This file is located in the Marvin Beans directory.)
  • Download the additional package for this Marvin Beans version: marvinbeans-lib-VERSION-signed.zip. The version ID of the additional package has to equal with the ID of the installed Marvin Beans package.
  • Create a backup about the lib sub-directory of your Marvin Beans package.
  • Extract the marvinbeans-lib-VERSION-signed.zip archive file into the Marvin Beans directory. Your extractor tool (e.g. unzip or WinZip) may ask confirmation to update all files by unwrapping. In this case, let it overwrite all. This operation will update jar files (substitute jars to signed ones) in the lib sub-directory of the installed Marvin Beans package.
